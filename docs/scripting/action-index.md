# CMG Action Index

Use this page when you need to find the right action family without reading the full reference from top to bottom. Detailed behavior, options, stdout, stderr, and failure modes live in [actions.md](actions.md).

| Area | Actions | Works In | GIF / Pointer Behavior |
| --- | --- | --- | --- |
| Navigation | [`navigate`, `goto`, `visit`](actions.md#navigate-goto-and-visit), [`reload`, `goBack`, `goForward`, waits](actions.md#reload-goback-goforward-waitforurl-waitforloadstate-waitfornetworkidle-and-waitfornavigation) | Direct scripts, `cmg run` | Navigation itself does not move the pointer. Later visual actions are recorded normally. |
| Waiting | [`waitForElement`](actions.md#waitforelement), [`waitForFunction`, `waitForTimeout`](actions.md#waitforfunction-and-waitfortimeout), [`pauseGif`](actions.md#pausegif), [`recordCheckpoint`](actions.md#recordcheckpoint), [`showPointer`, `hidePointer`](actions.md#showpointer-and-hidepointer), event waits | Direct scripts, `cmg run` | Non-visual by default. Use `step` or `caption` when the wait should be narrated in a GIF. Recording-only actions capture frames or metadata only while GIF recording is active and otherwise skip without injecting the virtual pointer. |
| Recording Defaults | [`recording`, `withRecording`](actions.md#recording-and-withrecording), [`gif`, `recordVideo`, `screencast`](actions.md#gif-recordvideo-and-screencast) | Direct scripts, `cmg run` | `recording` scopes choreography defaults without starting capture. `gif` and aliases create focused recordings unless command-level `--gif` is active. Reduced-motion and high-contrast-pointer presets remain inert without a recorder. |
| Accessibility Evidence | [`showKeystrokes`](actions.md#showkeystrokes), [`recording`, `gif`, and aliases`](actions.md#recording-and-withrecording) | Direct scripts, `cmg run` | Recording-only key labels, amplified focus rings, role/name callouts, and WCAG contrast warnings are added only during frame capture. Typed values are not shown. |
| GIF Diagnostics | [`recording`, `gif`, and aliases`](actions.md#recording-and-withrecording) with `debug=true` | Direct scripts, `cmg run` | A capture-only HUD and per-frame `.debug.json` sidecar expose action, scope, target, pointer, scroll, and timing evidence. No recorder means no HUD or pointer. |
| Narration | [`caption`](actions.md#caption), [`narrate`](actions.md#narrate), [`step`](actions.md#step) | Direct scripts, `cmg run` | Timed fades and automatic assertion/failure evidence are captured only with an active GIF. Narration never creates a virtual pointer in non-GIF runs. |
| GIF Privacy | [`maskGif`, `redactGif`, `redactText`](actions.md#maskgif-redactgif-and-redacttext), [`unmaskGif`, `unredactGif`](actions.md#unmaskgif-and-unredactgif) | Direct scripts, `cmg run` | Frame-only live masks protect recorded evidence without changing the page. Password inputs are masked by default; recording-only actions skip when no GIF is active. |
| Title Cards | [`intro`, `outro`](actions.md#intro-and-outro) | Direct scripts, `cmg run` | Recording-only full-viewport cards captured without the virtual pointer; scoped defaults can bookend a whole recording. |
| Timeline Editing | [`hideFromGif`, `cutGif`, `speedUpGif`, `slowDownGif`](actions.md#hidefromgif-cutgif-speedupgif-and-slowdowngif) | Direct scripts, `cmg run` | Hidden sections execute without frames or pointer UI; playback scopes scale encoded delays and nest safely. |
| Timeout Defaults | [`setDefaultTimeout`, `setDefaultNavigationTimeout`, `setDefaultAssertionTimeout`, scoped timeout blocks](actions.md#default-timeouts) | Direct scripts, `cmg run` | Non-visual. Use `caption` or `step` when the policy should be visible in a GIF. Scoped blocks restore previous defaults after child actions. |
| Pointer Input | [`click`](actions.md#click), [`tap`, `touchTap`](actions.md#tap-and-touchtap), [`hover`](actions.md#hover), [`dragAndDrop`](actions.md#draganddrop), [`mouseMove`, `mouseDown`, `mouseUp`](actions.md#mousemove-mousedown-and-mouseup) | Direct scripts, `cmg run` | Moves the virtual pointer and dispatches page-facing pointer/mouse events when GIF recording is active. Without GIF recording, no virtual pointer is injected. |
| Text And Keyboard | [`type`](actions.md#type), [`clear`](actions.md#clear), [`press`](actions.md#press), [`keyboardShortcut`, `hotkey`, `insertText`](actions.md#keyboardshortcut-shortcut-hotkey-keydown-keyup-and-inserttext) | Direct scripts, `cmg run` | Selector-based text actions move the pointer. Low-level keyboard actions do not move it. |
| Forms And Files | [`select`, `selectOption`](actions.md#select-and-selectoption), [`uploadFiles`, `setInputFiles`, `selectFile`](actions.md#uploadfiles-setinputfiles-and-selectfile), clipboard actions | Direct scripts, `cmg run` | Selector-based form actions move the pointer. File and clipboard actions are logged but non-visual. |
| Capture | [`html`](actions.md#html), [`screenshot`](actions.md#screenshot), [`screenshotPage`](actions.md#screenshotpage), [`printPdf`](actions.md#printpdf) | Direct scripts, `cmg run` | Element screenshots scroll the element into view. Page screenshots and PDF output do not move the pointer. Screenshot artifacts can use temporary CSS, masks, disabled animations, hidden carets, and page clips without changing GIF frame capture. |
| Assertions | [`assertText`](actions.md#asserttext), evaluated assertions, element assertion aliases, visual assertions | Direct scripts, `cmg run` | Pointer-aware assertions move to the target where that helps visual evidence. Non-visual assertions are included in output, reports, and traces. |
| Runtime Evaluation | [`evaluate`](actions.md#evaluate), selector evaluation, getters including [`computedStyle` and `property`](actions.md#textcontent-innertext-inputvalue-getattribute-computedstyle-and-property), init scripts, exposed functions, script/style tags | Direct scripts, `cmg run` | Evaluation and getters are non-visual. Values can be captured with `set name { ... }`. |
| Page And Environment | [`setViewport`, `highlight`](actions.md#highlight), `emulate`, media/geolocation/permissions/CSP/service worker controls | Direct scripts, `cmg run` | Environment changes are logged. `highlight` is visual and moves the GIF pointer to the target before drawing the callout. |
| Tabs And Popups | [`listTabs`, `openTab`, `waitForTab`, `waitForPopup`, `activateTab`, `closeTab`](actions.md#listtabs) | Direct scripts, `cmg run` | Tab management is non-visual unless followed by page actions in the active tab. |
| Events | Downloads, console capture/waits, page errors, dialogs, generic events | Direct scripts, `cmg run` | Event actions are explicit and logged. Dialogs are handled through dialog actions, not silently auto-removed. |
| Frames | [Frame actions](actions.md#frame-actions), [`frame`, `frameLocator`](actions.md#within) | Direct scripts, `cmg run` | Frame-scoped pointer actions move to the element's top-page coordinate inside the iframe. |
| Context And Storage | Context reset, browser contexts, storage state, local/session storage, cookies | Direct scripts, `cmg run` | Non-visual. Useful for setup, auth, isolation, and reproducible state. |
| Network | API requests, routes, intercepts, mocks, HAR, waits, WebSocket routes/waits, headers, credentials, proxy, offline | Direct scripts, `cmg run` | Non-visual. Reports and traces preserve step output and failure reasons. |
| Workers And Coverage | Worker listing/waits/evaluation/interception, coverage, tracing | Direct scripts, `cmg run` | Non-visual. Use traces/reports for inspection. |
| Files | [`readFile`, `fixture`, `writeFile`, `appendFile`, `expectFile`](actions.md#readfile-fixture-writefile-appendfile-and-expectfile) | Direct scripts, `cmg run` | Non-visual. Good for fixtures, assertions, and generated artifacts. |
| Structure | Suites, tests, parameterized `test.each`, hooks, [`step`](actions.md#step), [`gif`](actions.md#gif-recordvideo-and-screencast), imports, variables, generic [`expect` / `assert`](actions.md#expect-assert-softexpect-and-softassert), soft assertions, runtime [`skip`](actions.md#skip), control flow, loops, collection iteration, macros, `try`/`catch` | Direct scripts where applicable, `cmg run` | Structural actions do not move the pointer themselves. Pointer-aware child actions keep full GIF behavior. |
| Locators | [Locator support](actions.md#locator-support) | Direct scripts, `cmg run`, grouped CLI commands | Rich and provider-style locators resolve to the same element marker used by the GIF recorder. |

For command-line equivalents, start with [browser control](../commands/browser/control/index.md). Some actions, such as `set`, `within`, `frame` blocks, `gif` blocks, macros, and control flow, are script-only because they depend on script scope or child actions.
